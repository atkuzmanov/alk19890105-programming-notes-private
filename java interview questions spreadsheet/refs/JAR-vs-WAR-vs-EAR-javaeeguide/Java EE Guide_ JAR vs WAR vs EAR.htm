JAR vs WAR vs EAR

In Java world, you might have heard about different file formats such as JAR, WAR and EAR. Let's see what these files are and how it differs from each other.

When we develop an application we have multiple source files. We compile them and convert into byte code before executing these source files. In an IDE, it is acceptable for the developer to execute the application in this way. But, this approach is difficult and adds unwanted complexity while deploying the application.

Let's take an example of MySQL. We downloaded one single file and then installed the application in our local machine. Imagine if we have to take the source code and to install it properly or with compiled class files. Most of the applications will have 100s of files, if not in 1000s. It would become a mess if we try to copy all the files and follow a set of instructions to compile and run an application.

To avoid this complexity, the applications are packaged as a single file(Archive) with an additional XML file. This XML file informs the JVM about how to execute the application and other additional properties. There are three types of archives in Java world such as JAR, WAR and EAR. These file formats are nothing but ZIP files. You can change the extension of the file as .zip and extract the contents of the archive.

JAR (Java ARchive): This is the basic unit which is a collection of Java Class files and a deployment descriptor. If a main method is declared, a JAR file could be executed in any JRE environment without any Server. There is another scenario where EJB files could be packaged as a JAR file and included in an EAR file.

WAR (Web ARchive): This archive contains JSP, Servlets, HTML, CSS and any images needed to render the Web page. This will be deployed in a Web Server or a Web Container in an application server. This will have an XML file named web.xml which will give inputs such as Servlet to call for a specific URL, security etc.

EAR (Enterprise ARchive): This is a super set of above file formats. An EAR may consist of more than one WAR file and JAR file. There will be an XML file called application.xml which is the deployment descriptor for this file. EAR file contains configuration such as application security role mapping, EJB reference mapping and context root url mapping of web modules.

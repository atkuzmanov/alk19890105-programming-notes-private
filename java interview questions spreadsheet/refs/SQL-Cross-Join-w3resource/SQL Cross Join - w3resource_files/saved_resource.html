                                          w3resource Mod Dado • 6 years ago

                                          Since in many a use cases, using a ON clause or even subquery will do what you want to achieve with CROSS JOIN. Moreover, a wrong use may cause you return erroneous data. Because of this, application of CROSS JOIN is limited in real world. Having said that, in certain reports or (may be in an update statement) you may use CROSS JOIN
                                          to return m*n rows from A and B table, where m is the number of rows present in A and n is number of rows present in B table.

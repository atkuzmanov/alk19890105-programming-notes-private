    First time when i saw the output, got confused why the sysout in main method got executed first. since there is no explanation for this, i thought of clarifying here.

    The main point that has to be noted in Multi threading is, "At a time only one thread gets executed" which is the reason for the above output.

    When we call the line t.start(); in below code, the thread goes into the runnable state (i.e any time this thread can get executed, this is the responsibility of thread scheduler to relinquish the CPU to this thread when deserved)

    Here currently the CPU is running the main method so the " System.out.println("Main method executed by main thread");" line gets executed first and the thread "t" gets executed once the main thead is done.

    put Thread.sleep(1000); after t.start(); in the code below. for better clarity :)

    // Java program to illustrate defining Thread
    // by extending Thread class

    // Here we cant extends any other class
    class Test extends Thread
    {
    public void run()
    {
    System.out.println("Run method executed by child Thread");
    }
    public static void main(String[] args)
    {
    Test t = new Test();
    t.start();
    System.out.println("Main method executed by main thread");
    }
    }

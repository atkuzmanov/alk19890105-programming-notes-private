JPA provides you the ability to code generically at a higher layer without worrying about the underlying database and/or ORM. JPA works at a layer above ORMs such as Hibernate.

So, you have JPA > Hibernate(or some other ORM like iBatis) > JDBC(Hibernate internally uses JDBC) > database(Oracle/MySql etc).

JPA can connect to any ORM provider of your choice which provides a “Provider” to connect with it. You code against JPA APIs. Later you can change the database whenever you want ‘and’ ORM as well. So, you can have -

JPA > Hibernate > Oracle

or

JPA>iBatis> Oracle
